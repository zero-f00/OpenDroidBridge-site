macOS file transfer for Android-based devices

OpenDroidBridge

MacからAndroid端末やMeta Questへ、必要なファイルを安全に移動。USB接続、復元できる削除、同名ファイル保護、スクリーンショット保存まで一つにまとめます。

使い始める 接続手順を見る
Android phones Meta Quest ADB over USB macOS 14+

Core workflow

ファイル転送で怖いところを、最初から見える形に。

端末の状態、現在の場所、空き容量、操作リスクを確認しながら、Macへ保存・端末へ追加・新規フォルダ作成を進められます。

見つける

DCIM、Pictures、Movies、Download、Quest録画など、よく使う場所へすばやく移動できます。

転送する

選択したファイルやフォルダをMacへ保存。Macから端末側フォルダへドラッグして追加できます。

戻せる

削除や上書き前の項目を復元エリアへ残し、誤操作に気づいたあとでも確認し直せます。

Real device preview

Questの録画やスクリーンショットも、Mac側から整理。

  • 接続中のAndroid端末とMeta Questを並べて表示
  • Quest向けのOculusフォルダ、録画フォルダへショートカット
  • 端末ストレージの空き容量と現在のパスを常に表示
  • USBデバッグ未許可やオフライン状態も診断しやすい表示

Safety model

置き換えも削除も、いきなり消さない。

同名ファイルを送るときは、置き換える・両方残す・スキップを選択。置き換え前の項目や削除した項目は、復元できる場所から確認できます。

大切なファイルを守るための安全設計です。

OpenDroidBridgeは、端末内ファイル操作をユーザーの明示的な操作として扱い、復元できる処理を既定にしています。

Quest diagnostics

Quest接続の確認とスナップショット取得にも対応。

Quest Display画面では、ADB接続、Quest画面スナップショット、実機情報、ADB forwardの準備状態を確認できます。

Quest側の録画やスクリーンショット整理に加えて、接続状態を確認したいときにも使えます。

Optional AI CLI assistant

AIに聞ける。でも、操作はあなたの確認で。

Codex CLI、Claude Code、Ollama、LM Studioなど、手元のAI CLIやローカルLLMに現在の端末状態を渡して、ファイル整理や復元の手順を相談できます。

文脈を渡す

選択中の端末種別、接続状態、現在フォルダの件数、操作リスク、ユーザーが明示的に追加したMac側参照だけを相談材料にします。

CLI案をもらう

AIはOpenDroidBridgeのUI操作やopendroidctlコマンドを提案します。raw adbより安全な操作手順を優先するプロンプトで実行されます。

勝手に実行しない

AI連携は任意です。AIがファイル転送、削除、ADBコマンドを自動実行することはなく、実際の操作はアプリ側の確認フローに残ります。

Requirements

必要環境

Mac

macOS 14以降。

Android / Meta Quest

開発者向けオプションとUSBデバッグを有効化したAndroid端末、または開発者モードを有効化したMeta Quest。

ADB

Android platform-toolsに含まれるadbが必要です。Android Studioを入れている場合は通常同梱されています。
